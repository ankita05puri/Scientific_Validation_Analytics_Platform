# Scientific Intelligence Product Vision

This document is the product constitution for Scientific Intelligence and SVAP. When a design, product, or implementation choice is unclear, use this vision as the reference point.

## Our Mission

We are building the world's leading Scientific Intelligence platform for experimental science.

Our goal is not to create another laboratory application.

Our goal is to build the operating system for scientific reasoning.

Every feature, workflow, and interface should help scientists think more clearly, preserve scientific knowledge, and make better scientific decisions.

## The Problem We Solve

Today's scientific work is fragmented.

Experiments are scattered across:

- laboratory instruments
- spreadsheets
- notebooks
- statistical software
- presentations
- reports
- disconnected applications

The files survive.

The scientific reasoning often does not.

Scientists repeatedly reconstruct the same story for lab meetings, reviewers, publications, and regulatory submissions.

Scientific knowledge should not disappear when a scientist closes their laptop or leaves the laboratory.

## Our Philosophy

We do not organize files.

We do not organize reports.

We organize scientific reasoning.

Every experiment should become a reusable scientific asset that preserves:

- scientific question
- experimental design
- laboratory context
- experimental evidence
- statistical analysis
- scientific interpretation
- scientific decisions
- controlled publications
- complete traceability

The experiment becomes the source of truth.

Everything else should be generated from it.

## The Five Scientific Systems

Every application we build should support one or more of these systems.

### 1. Design

Plan experiments.

Define protocols.

Specify acceptance criteria.

Question: What should we do?

### 2. Study

Execute experiments.

Capture evidence.

Ensure data quality.

Track scientific progress.

Question: What happened?

### 3. Understand

Transform experimental data into understanding.

Perform statistics.

Visualize results.

Apply machine learning and AI.

Question: What do the data tell us?

### 4. Interpret

Review evidence.

Support scientific judgment.

Capture reviewer decisions.

Maintain traceability.

Question: What do the results mean?

### 5. Share

Publish controlled reports.

Generate regulatory evidence.

Preserve institutional scientific knowledge.

Communicate trusted scientific conclusions.

Question: How do we preserve and communicate scientific knowledge?

## Product Principles

Every product decision should satisfy these principles.

### Scientific-first

Design for scientists, not generic business users.

Scientific workflows take priority over generic dashboard patterns.

### Preserve reasoning

Capture not only data, but the reasoning and decisions behind it.

Future scientists should understand why, not only what.

### One source of truth

Every experiment exists once.

Reports, presentations, AI summaries, regulatory documents, and publications are generated from the same scientific object.

Never require scientists to recreate the same information.

### AI is a capability, not the product

Scientific Intelligence is the product.

AI assists scientific work but never replaces scientific judgment.

Every AI output should remain transparent and traceable.

### Simplicity over features

Every screen should answer one question.

Remove anything that does not help scientists make better decisions.

Prefer clarity over customization.

### Build systems, not pages

Applications should feel like connected parts of one scientific operating system.

The user should move naturally from:

Design -> Study -> Understand -> Interpret -> Share

without changing mental models.

## SVAP

Scientific Validation Analytics Platform (SVAP) is the first application built on Scientific Intelligence.

Its purpose is to prove that structured scientific reasoning can improve how validation studies are designed, executed, reviewed, and published.

Current lifecycle:

Programs -> Execution -> Review -> Reports

SVAP is not the destination.

It is the foundation for every future Scientific Intelligence application.

## Long-Term Vision

Scientific Intelligence should become the trusted foundation for experimental science.

Whether a scientist is validating a diagnostic assay, optimizing a semiconductor process, discovering a catalyst, or developing a new material, the platform should help them preserve context, understand evidence, make better decisions, and transform experiments into reusable scientific knowledge.

## Design Test

Before implementing any feature, ask:

1. Does this help scientists think more clearly?
2. Does it preserve scientific reasoning?
3. Does it reduce repetitive scientific work?
4. Does it make experiments more reusable by humans and AI?
5. Does it fit naturally into one of the five scientific systems?

If the answer to these questions is no, reconsider whether the feature belongs in Scientific Intelligence.
